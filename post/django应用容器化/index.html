生成依赖文件

在pycharm中，打开Terminal，输入如下命令

1

pip freeze>requirements.txt

FROM python:3.7
LABEL maintainer="maisy.vip"
RUN apt-get update \
    && apt-get install -y --no-install-recommends \
        postgresql-client \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /usr/src/app
COPY requirements.txt ./
RUN pip install -r requirements.txt
COPY . ./

EXPOSE 8000
CMD ["python", "manage.py", "runserver", "0.0.0.0:8000"]
  • FROM指定当前镜像的基础镜像层
  • LABEL指定了当前镜像的维护者
  • RUN 运行命令安装所需要的数据库或者其他依赖软件
  • WORKDIR 为后续未执行的命令指定工作目录
  • COPY 将相关文件从当前上下文复制到当前镜像中
  • RUN pip 安装requirements.txt文件中的依赖
  • COPY 其他复制操作
  • EXPOSE 指定端口以及完成相关设置，用来提供web服务
  • CMD 只能有一个，启动容器的时候会运行该命

构建镜像

在项目的路径下，运行命令

1

docker image build -t equipment:latest .
  • equipment 为镜像的名称，latest为标签
  • 不好忽略了结尾的点.
  • 构建完成后，docker iamges即可看到刚才构建的镜像

运行容器

1

docker run --restart=always -d -p 8000:8000 equipment
  • –restart=always 自动重启容器
  • -d指定容器在后台运行
  • -p 映射端口，左边为本地端口，右边为容器端口

# 参考官方教程  
# https://docs.djangoproject.com/zh-hans/2.2/howto/static-files/

1. 将 STATIC_ROOT 配置成你喜欢的目录，在这个目录提供服务，例如:

STATIC_ROOT = "/var/www/example.com/static/"

2. 运行 collectstatic 管理命令:

$ python manage.py collectstatic

这将会把静态目录下的所有文件拷贝至 STATIC_ROOT 目录。

3. 选一个 Web 服务器为这些文件提供服务。 文档 部署静态文件 介绍了静态文件的常见部署策略。
